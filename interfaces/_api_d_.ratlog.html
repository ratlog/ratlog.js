Interface Ratlog

Ratlog is the main logging function.

Hierarchy

  • Ratlog

Callable

  • __call(message: Stringable, fields?: object, ...tags: Stringable[]): void
    • Defined in api.d.ts:41

    Mostly you are simply using Ratlog as a function that you call to log something.

    Ratlog takes a message, an object of fields and a rest of tags.

    No argument is required, but the typings force you to at least provide a message. That is a good practise because an empty log line is mostly meaningless.

    fields and tags are optional:

    log('hey there')

    If you want to add tags but no fields, you cannot ommit the fields but you could do something like this:

    log('msg', null, 'mytag')

    However, the intend can be communicated better by writing it as:

    log.tag('mytag')('msg')
